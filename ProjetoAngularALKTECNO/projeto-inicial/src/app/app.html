Criando projeto em Angular.

Pessoal, no vídeo anterior nós fizemos a instalação do Angular CLI e com ele nós vamos poder criar a estrutura do nosso projeto. Além disso, nós podemos executar esse projeto e ver ele sendo executado no navegador, que é o que nós vamos fazer neste vídeo.

Vamos criar uma estrutura inicial, fazer uma breve análise de alguns arquivos e pastas e também inicializar esse projeto para ver ele sendo executado no navegador. Beleza?

Como já temos a biblioteca @angular/cli instalada globalmente na máquina, podemos utilizar o comando ng para criar a estrutura do projeto.

Por exemplo: ng new projeto-inicial. Dessa forma ele cria toda a estrutura e define o nome do projeto.

Caso o Angular CLI não estivesse instalado globalmente, poderíamos utilizar o npx, que baixa e executa automaticamente a versão mais recente do Angular CLI sem ocupar espaço permanente na máquina.

A vantagem do npx é sempre usar a versão mais atual do Angular. A desvantagem é o tempo de download sempre que o comando for executado.

Durante a criação do projeto, o Angular pergunta se desejamos adicionar rotas e qual tipo de estilo será utilizado (CSS, SCSS, SASS ou LESS). Após isso, ele baixa todas as dependências e cria o node_modules e o package.json.

O arquivo package.json é responsável pelo gerenciamento das dependências do projeto, incluindo o Angular, RxJS, Zone.js e ferramentas de testes como Karma e Jasmine.

Outro arquivo importante é o angular.json, que contém todas as configurações do projeto, como assets, estilos globais e processo de build.

A pasta src é o coração do projeto. É nela que ficam os componentes, estilos, imagens, ícones, testes unitários e todo o código da aplicação.

Dentro da pasta app fica o componente principal (AppComponent), que possui um template HTML, um arquivo de estilo e um arquivo TypeScript com a lógica do componente.

O Angular inicializa a aplicação a partir do arquivo main.ts, que faz o bootstrap do módulo principal, carregando o primeiro componente que será renderizado no navegador.

Para executar o projeto, utilizamos o comando ng serve, que cria um servidor local e disponibiliza a aplicação normalmente em http://localhost:4200.

Durante o processo de build, o Angular transforma o código TypeScript em JavaScript, que é o formato entendido pelo navegador.

Os componentes são renderizados no DOM por meio de seus seletores, que são utilizados dentro do arquivo index.html.

Por fim, essa foi uma visão geral da criação, estrutura e inicialização de um projeto Angular. Alguns conceitos como bootstrap e módulos podem parecer confusos no início, mas com a prática tudo vai ficando mais claro.

- Shortcuts

- "Ctrl + v" (colar o texto copiado)

- "Ctrl + s" (salvar um arquivo)

- "Ctrl + z" (voltar a alteração)

- "Ctrl + ;" (comentar uma linha)

- "Ctrl + '" (abrir o terminal)

- "Ctrl + b" (abrir e fechar o explorer)

- "Ctrl + f" (pesquisar um texto no arquivo)

- "Ctrl + p" (pesquisar um arquivo pelo nome)

- "Alt + Shift + Seta-Baixo" (duplicar linha)

- "Ctrl + ." (mostrar sugestões de quick fix)

- "Ctrl + Espaço" (mostrar propriedades de um objeto)
