Projeto Angular Componentes

Bem-vindo ao nosso sistema de cards!

O que foi apresentado no vídeo?

O vídeo apresenta a criação de cards em um projeto Angular, discutindo a importância de avaliar se vale a pena criar vários componentes quando o layout é semelhante e apenas textos ou cores mudam.
O objetivo principal é mostrar conceitos fundamentais da estrutura do Angular e como organizar melhor os elementos da interface para manter o código limpo e reutilizável.

O projeto foi configurado usando Angular CLI versão 16, com uma estrutura simples, sem rotas e utilizando CSS para estilização. Após a configuração, o projeto foi executado com o comando ng serve, permitindo visualizar a aplicação no navegador através do endereço local padrão.
Essa etapa demonstra o fluxo básico para iniciar o desenvolvimento e testar alterações rapidamente. Durante o desenvolvimento, foi criado um componente de card com o comando de geração do Angular CLI, que automaticamente produz os arquivos necessários como HTML, CSS, TypeScript e testes.
Foi explicado que cada componente é definido por uma classe TypeScript com o decorator @Component, contendo propriedades essenciais como selector,
template e estilos, permitindo sua reutilização como uma tag personalizada dentro da aplicação. Por fim, o vídeo reforça a importância da organização entre componentes e módulos, destacando que componentes precisam ser declarados no módulo para funcionar corretamente e que módulos externos devem ser importados separadamente. A conclusão reforça a compreensão da base estrutural do Angular e prepara o terreno para etapas futuras,
onde será desenvolvido o layout do card e analisada a melhor abordagem para reutilização de componentes.

Bem-vindo ao nosso sistema de cards!
